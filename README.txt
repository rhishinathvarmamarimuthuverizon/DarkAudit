🚀 Installation & Run Options
Option 1: Standalone EXE (Recommended for Windows)
No Java installation needed — bundled JRE included.

Double-click DarkAudit.exe
Option 2: JAR File (Cross-platform)
Requires JDK 11 or higher.

java -jar DarkAudit.jar
Option 3: Launcher Scripts
Windows:

DarkAudit-GUI.bat          # GUI mode (auto-finds Java)
DarkAudit-CLI.bat --help   # CLI mode
Linux / macOS:

./darkaudit.sh             # GUI mode
./darkaudit-cli.sh --help  # CLI mode
💻 CLI Usage
DarkAudit supports fully headless operation for automation and CI/CD pipelines.

Basic Scan
java -jar DarkAudit.jar --scan app.apk
Specify Output Format
java -jar DarkAudit.jar --scan app.apk --format html
java -jar DarkAudit.jar --scan app.apk --format pdf
java -jar DarkAudit.jar --scan app.apk --format sarif
java -jar DarkAudit.jar --scan app.apk --format excel
java -jar DarkAudit.jar --scan app.apk --format word
java -jar DarkAudit.jar --scan app.apk --format pptx
java -jar DarkAudit.jar --scan app.apk --format csv
java -jar DarkAudit.jar --scan app.apk --format json
Specify Output Path
java -jar DarkAudit.jar --scan app.apk --format sarif --output results.sarif.json
CLI Reference
Usage:
  java -jar DarkAudit.jar                                          Launch GUI
  java -jar DarkAudit.jar --scan <apk>                             Scan APK (headless, JSON default)
  java -jar DarkAudit.jar --scan <apk> --format <fmt>              Scan with specific format
  java -jar DarkAudit.jar --scan <apk> --format <fmt> --output <path>
  java -jar DarkAudit.jar --server [--port 8089]                   Start REST API server
  java -jar DarkAudit.jar --help                                   Show help

Formats: json, html, sarif, pdf, excel (xls), word (docx), pptx (powerpoint), csv

Exit Codes:
  0 — PASS (no critical/high findings)
  1 — ERROR (invalid input, runtime failure)
  2 — FAIL (critical or high severity findings detected)
🌐 REST API Server
Start the built-in HTTP server for programmatic access:

java -jar DarkAudit.jar --server --port 8089
Endpoints
Method	Endpoint	Description
GET	/api/health	Health check — returns {"status":"ok","version":"2.0.0"}
GET	/api/rules	List all active security rules
POST	/api/scan	Upload APK binary, returns JSON findings
Examples
# Health check
curl http://localhost:8089/api/health

# List rules
curl http://localhost:8089/api/rules

# Scan APK
curl -X POST --data-binary @app.apk http://localhost:8089/api/scan
Response Format

      "cwe": "CWE-798"
    }
  ],
  "total": 1
}

📝 Custom Rules (YAML)
Create custom security rules without modifying source code. Place .yaml files in ~/.darkaudit/rules/:

id: CUSTOM-001
name: Hardcoded API Endpoint
severity: HIGH
pattern: "https?://[a-zA-Z0-9.-]+/api/"
targets: [JAVA_SOURCE]
impact: Hardcoded API endpoints can be extracted by attackers through reverse engineering.
fix: Use BuildConfig fields or encrypted configuration files.
cvss: 5.3
id: CUSTOM-002
name: Sensitive Logging in Release
severity: CRITICAL
pattern: "Log\\.(d|v|i)\\(.*(password|token|secret|key)"
targets: [JAVA_SOURCE]
impact: Sensitive data logged in debug/verbose level may be captured via logcat.
fix: Remove all sensitive data from log statements or use ProGuard to strip logging.
cvss: 7.5
Supported fields: id, name, severity (CRITICAL/HIGH/MEDIUM/LOW/INFO), pattern (regex), targets (JAVA_SOURCE/MANIFEST/RESOURCE), impact, fix, cvss

🔄 CI/CD Integration
GitHub Actions
name: DarkAudit Security Scan
on: [push, pull_request]

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Run DarkAudit
        run: |
          java -jar DarkAudit.jar --scan app/build/outputs/apk/release/app-release.apk \
            --format sarif --output results.sarif.json

      - name: Upload SARIF to GitHub Security
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: results.sarif.json

      - name: Upload Report Artifact
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: security-report
          path: results.sarif.json
GitLab CI
security-scan:
  stage: test
  image: openjdk:17-slim
  script:
    - java -jar DarkAudit.jar --scan app.apk --format json --output gl-sast-report.json

    }
}
